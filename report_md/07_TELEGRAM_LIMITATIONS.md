# Telegram Limitations

## Telegram API Hard Limits

| Limit | Value | Impact |
|-------|-------|--------|
| Message frequency per chat | 30 msg/sec | Cannot exceed with bot actions |
| Webhook setup | One URL per bot | Single endpoint for all updates |
| Bot privacy mode | Cannot see other bots' messages | Limits cross-bot detection |
| Message edit window | 48 hours | Cannot act on edited messages after window |
| Group size | 200,000 members | Large groups hit resource limits |

## Privacy Mode Constraints

**Location:** `SECURITY_MODEL.md:108-115`

### What Bots Cannot See (Privacy Mode)

- Messages from private groups the bot is not admin in
- Messages older than when bot was added (sometimes)
- Other bots' messages (always)

**Workaround:** Bot must be made admin with "Allow reading messages" permission.

## Webhook Limitations

### Secret Token

- `X-Telegram-Bot-Api-Secret-Token` header for webhook verification
- Token is user-set, not generated by Telegram
- If compromised, attacker can send fake updates

**Current Implementation:** Checked at `webhook.ts:59-70` ✅

### Update Frequency

- Telegram sends updates at most 100/sec across all chats
- Under heavy load, updates may be batched or dropped

**Impact:** Real-time protection may have gaps during DDoS-style attacks.

### Telegram Error Codes

| Code | Meaning | Handling |
|------|---------|----------|
| 429 | Too Many Requests | Retry with exponential backoff |
| 400 | Bad Request | Do not retry |
| 403 | Bot was blocked | Do not retry |
| 403 | User is deactivated | Do not retry |
| 400 | Message to delete not found | Already deleted, idempotent success |
| 400 | Not enough rights | Bot needs admin rights |

## Moderation Action Limitations

### Delete Message
- Bot must be admin with delete rights
- Only works on messages within 48 hours old
- Cannot delete messages from other bots if privacy mode enabled

### Restrict/Ban
- Bot must be admin with restrict rights
- Cannot restrict or ban group creators
- Cannot restrict/bot itself

### Lockdown
- Bot must be admin with all permissions
- Removes ability for anyone to message (including admins if misconfigured)

## Message Entity Parsing

**Location:** `packages/detection-engine/src/text-normalizer.ts`

### Known Gaps

- URLs embedded in code blocks may not be detected
- Mixed-script URLs (Latin + Cyrillic) detection limited
- Inline media captions with links not parsed

---

## Broadcast Limitations

For cross-group threat intelligence:
- Bot can only message groups it is admin in
- No way to send "warning" messages to unrelated groups
- Cross-group coordination requires separate admin consent per group

---

## File Size Limits

- Photo: 10MB max
- Video: 50MB max
- Documents: 50MB max

**Media Analysis Gap:** Large files cannot be analyzed for malicious content in current implementation.
