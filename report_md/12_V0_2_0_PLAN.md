# v0.2.0 Plan

## Theme: Autonomous Agent Foundation

## Goals

1. Close all v0.1.0 security gaps
2. Implement LLM-powered message analysis
3. Add user behavior memory
4. Establish cross-group threat intelligence
5. Validate performance at scale

---

## Sprint 1: Auth & Security (2 weeks)

### Authentication

| Item | Description | Owner |
|------|-------------|-------|
| Telegram Login Widget | Login page + backend verification | API |
| JWT Session Management | Issue/refresh/revoke tokens | API |
| RBAC Middleware | Group admin verification | API |
| Per-IP Rate Limiting | Block excessive API consumers | API |

### Webhook Security

| Item | Description | Owner |
|------|-------------|-------|
| Replay Protection | Update ID deduplication in Redis | API |
| Secret Rotation | Allow secret rotation without downtime | API |

## Sprint 2: AI Integration (2 weeks)

### LLM Classifier

| Item | Description | Complexity |
|------|-------------|------------|
| Anthropic SDK | Claude API integration | Medium |
| Prompt Caching | Cache conversation context | Medium |
| Fallback Classifier | Heuristic when AI unavailable | Low |
| Async Analysis Queue | Non-blocking AI calls | Medium |

### Classification Types

| Type | Prompt Strategy |
|------|-----------------|
| Spam | Zero-shot with examples |
| Scam | Pattern matching + AI check |
| Harassment | Sentiment analysis |
| Impersonation | Username similarity scoring |

## Sprint 3: User Memory (2 weeks)

### Database Extensions

| Item | Description |
|------|-------------|
| `user_behaviors` table | Per-user threat score history |
| `trust_scores` table | Graduated trust tracking |
| `false_positives` table | Feedback for self-improvement |

### Memory Implementation

| Item | Description |
|------|-------------|
| Behavior accumulation | Track violations per user over time |
| Trust graduation | New user → trusted after X days without violation |
| Anomaly detection | Flag users whose behavior changes suddenly |
| Admin feedback loop | Allow admins to mark false positives |

---

## Sprint 4: Cross-Group Intel (2 weeks)

### Threat Sharing

| Item | Description |
|------|-------------|
| Domain blocklist sharing | Share blocked domains across groups |
| User blocklist sharing | Share banned user Telegram IDs |
| Raid correlation | Detect coordinated raids across groups |
| Alert aggregation | Correlate alerts from multiple groups |

### Privacy Considerations

- Groups opt-in to threat sharing
- Only Telegram IDs shared, not user data
- Domain blocklists are public good
- Audit log of what was shared
